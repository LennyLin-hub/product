# 注塑排程系统关键配置流程

本文档说明系统中与“计划 - 派工 - 执行 - 异常 - 完工”相关的核心数据模型与配置口径，适合作为排程方案设计说明。

> 简化工序：SETUP → INJECT → POST_QC_PUTAWAY  
> 备料备模：作为任务最早可开始时间 `earliest_start` 的前置约束，不占用机台时间块。

---

## 0) A2 产能算法（推荐口径，供 INJECT 工序计算标准时长）
设：
- `cycle_time_sec`：节拍（秒/循环）
- `cavity`：型腔数（件/循环）
- `yield_rate`：良率（0~1）
- `utilization`：有效利用率（0~1，可先固定 1 或维护默认值）
- `batch_qty`：批次数量（件）

计算：
- `cycles_per_hour = 3600 / cycle_time_sec`
- `theoretical_output_per_hour = cycles_per_hour * cavity * yield_rate`
- `actual_output_per_hour = theoretical_output_per_hour * utilization`
- `run_time_hours = batch_qty / actual_output_per_hour`
- `run_time_min = run_time_hours * 60`

> 说明：SETUP 工序时长由“基础准备 + 换型”组成；POST 工序时长建议用单位工时（sec/件）或固定+变动。

## 1) 计划（Plan）：订单 → 路线 → 批次 → 任务

### 1.1 CustomerOrder（订单头）
用于订单状态跟踪、交期与优先级驱动排程。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| order_id | 订单ID（主键） | VARCHAR(32) |
| customer_id | 客户ID | VARCHAR(32) |
| due_date | 交期（用于 EDD） | DATE |
| priority | 优先级（插单/加急） | INT |
| status | NEW/CONFIRMED/IN_PRODUCTION/DONE | VARCHAR(20) |
| created_at | 创建时间 | DATETIME |

---

### 1.2 OrderLine（订单行）
一个订单可含多个产品行。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| order_line_id | 订单行ID（主键） | VARCHAR(32) |
| order_id | 所属订单（外键） | VARCHAR(32) |
| product_id | 产品/SKU | VARCHAR(32) |
| qty | 需求数量（件） | INT |

---

### 1.3 ProductRoute（产品路线头）
用于控制产品工艺版本。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| route_id | 路线ID（主键） | VARCHAR(32) |
| product_id | 产品ID | VARCHAR(32) |
| version | 工艺/图纸版本 | VARCHAR(20) |
| is_active | 是否启用（1启用） | TINYINT |

---

### 1.4 RouteOperation（路线工序）
将简化流程固化为 3 个工序：SETUP/INJECT/POST_QC_PUTAWAY。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| op_id | 工序定义ID（主键） | VARCHAR(32) |
| route_id | 所属路线ID | VARCHAR(32) |
| op_code | SETUP/INJECT/POST_QC_PUTAWAY | VARCHAR(32) |
| sequence | 顺序号（1/2/3） | INT |
| eligible_resource_rule | 可用资源规则引用 | VARCHAR(64) |
| std_time_model | 标准工时模型引用 | VARCHAR(64) |
| queue_policy | 排队策略（如 EDD） | VARCHAR(32) |

---

### 1.5 ProductionBatch（生产批次）
用于拆分订单、便于插单与追溯（强烈建议）。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| batch_id | 批次ID（主键） | VARCHAR(32) |
| order_line_id | 来源订单行 | VARCHAR(32) |
| batch_qty | 批次数量（件） | INT |
| status | PLANNED/RELEASED/IN_PROCESS/DONE | VARCHAR(20) |
| planned_start | 计划开工（可选） | DATETIME NULL |
| planned_end | 计划完工（可选） | DATETIME NULL |

---

### 1.6 OperationTask（工序任务）
由批次 + 路线自动生成 3 个任务，并写入预计时长 `std_duration_min`。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| task_id | 任务ID（主键） | VARCHAR(32) |
| batch_id | 所属批次 | VARCHAR(32) |
| op_code | SETUP/INJECT/POST_QC_PUTAWAY | VARCHAR(32) |
| sequence | 工序顺序（1/2/3） | INT |
| std_duration_min | 预计时长（分钟） | INT |
| earliest_start | 最早可开始时间（受备料/备模/前置任务影响） | DATETIME |
| status | READY/SCHEDULED/RUNNING/PAUSED/DONE/CANCELLED | VARCHAR(20) |

---

### 1.7 TaskDependency（任务依赖）
用于强制工序先后：SETUP → INJECT → POST。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| pre_task_id | 前置任务 | VARCHAR(32) |
| post_task_id | 后续任务 | VARCHAR(32) |

> 可用 `(pre_task_id, post_task_id)` 作为联合主键。

---

### 1.8 ScheduleJob（排程任务记录）
用于记录异步全量排程的提交状态、执行进度和最终结果。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|---|
| job_id | 任务ID（主键） | VARCHAR(32) |
| job_type | 任务类型（当前主要为 FULL） | VARCHAR(20) |
| status | PENDING/RUNNING/SUCCESS/FAILED | VARCHAR(20) |
| assignment_start | 排程开始时间基准 | DATETIME |
| started_at | 实际开始时间 | DATETIME |
| finished_at | 实际完成时间 | DATETIME |
| total_task_count | 任务总数 | INT |
| batch_count | 批次数量 | INT |
| scheduled_task_count | 已排程任务数 | INT |
| processed_task_count | 已处理任务数 | INT |
| progress_percent | 当前进度百分比 | INT |
| error_message | 错误信息 | VARCHAR(500) |

## 2) 派工（Dispatch）：资源模型 + 排程结果

### 2.1 Resource（统一资源主表）
设备/模具/人员/工位统一建模，便于冲突检测与利用率统计。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| resource_id | 资源ID（主键） | VARCHAR(32) |
| resource_type | MACHINE/MOLD/PERSON/WORKSTATION | VARCHAR(20) |
| name | 资源名称 | VARCHAR(64) |
| status | AVAILABLE/BUSY/DOWN/MAINTENANCE/OFFSHIFT | VARCHAR(20) |
| calendar_id | 班次日历ID | VARCHAR(32) |
| org_unit | 车间/班组/部门 | VARCHAR(32) |

---

### 2.2 Calendar（班次/日历）
控制资源可用时间窗口。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| calendar_id | 日历ID（主键） | VARCHAR(32) |
| workday_pattern | Mon-Sat 等 | VARCHAR(20) |
| shift_start | 班次开始（HH:MM） | VARCHAR(5) |
| shift_end | 班次结束（HH:MM） | VARCHAR(5) |
| breaks | 休息时间段数组（可选） | JSON |

---

### 2.3 Machine（注塑机扩展）
用于能力标注与默认换模基准时间。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| machine_id | 对应 Resource.resource_id | VARCHAR(32) |
| tonnage | 锁模力（能力约束可选） | INT |
| default_setup_time_min | 默认换模基准时间（分钟） | INT |
| supported_materials | 支持材料标签（可选） | JSON |

---

### 2.4 Mold（模具扩展）
建议将模具作为“独占资源”，避免同一模具被排到两台机同时生产。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| mold_id | 对应 Resource.resource_id | VARCHAR(32) |
| mold_code | 模具业务编号 | VARCHAR(32) |
| cavity | 型腔数（A2公式参数） | INT |
| mold_status | AVAILABLE/IN_USE/REPAIR/MAINTENANCE | VARCHAR(20) |
| next_maint_due | 下次保养到期（可选） | DATE NULL |

---

### 2.5 MachineMoldCompatibility（机模兼容）
硬约束：不兼容则不能排。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| machine_id | 注塑机ID | VARCHAR(32) |
| mold_id | 模具ID | VARCHAR(32) |
| is_compatible | 1兼容/0不兼容 | TINYINT |

> 可用 `(machine_id, mold_id)` 作为联合主键。

---

### 2.6 ResourceCapability（能力矩阵）
控制“某资源能做哪些产品的哪些工序”。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| resource_id | 资源ID | VARCHAR(32) |
| op_code | 工序编码 | VARCHAR(32) |
| product_id | 产品ID | VARCHAR(32) |
| is_enabled | 是否启用 | TINYINT |
| priority_weight | 同等可用时优先选择（可选） | INT |

> 可用 `(resource_id, op_code, product_id)` 作为联合主键。

---

### 2.7 ProductMoldParam（A2 产能参数表）
建议用 `(product_id, mold_id)` 维护节拍与良率，避免“同产品换不同模导致参数不同”混乱。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| product_id | 产品ID | VARCHAR(32) |
| mold_id | 模具ID | VARCHAR(32) |
| cycle_time_sec | 节拍（秒/循环） | DECIMAL(10,2) |
| cavity | 型腔数（件/循环） | INT |
| yield_rate | 良率（0~1） | DECIMAL(5,4) |
| utilization | 有效利用率（0~1，默认可为1） | DECIMAL(5,4) |

> 可用 `(product_id, mold_id)` 作为联合主键。

---

### 2.8 ChangeoverRule（换型规则）
用于 SETUP 工序计算换模/换料/换色时间（最小但有效）。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| rule_id | 规则ID（主键） | VARCHAR(32) |
| same_mold_time_min | 同模切换时间 | INT |
| different_mold_time_min | 换模时间 | INT |
| material_change_extra_min | 换料附加时间 | INT |
| color_change_extra_min | 换色附加时间 | INT |

---

### 2.9 TaskAssignment（派工/排程结果）
排程的核心输出：每个任务在资源上的时间块。  
SETUP/INJECT 必须同时占用 `machine_id + mold_id`，POST 用工位/人员。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| assignment_id | 派工记录ID（主键） | VARCHAR(32) |
| task_id | 任务ID | VARCHAR(32) |
| machine_id | 注塑机（SETUP/INJECT必填） | VARCHAR(32) NULL |
| mold_id | 模具（SETUP/INJECT必填） | VARCHAR(32) NULL |
| person_id | 人员（可选约束） | VARCHAR(32) NULL |
| workstation_id | 工位（POST可用） | VARCHAR(32) NULL |
| start_time | 计划开始时间 | DATETIME |
| end_time | 计划结束时间 | DATETIME |
| sequence_on_resource | 资源上的顺序号（甘特图） | INT |
| changeover_time_min | 换型时间（SETUP统计/解释） | INT |

---

### 2.10 SchedulePhase（排程阶段枚举）
用于异步排程任务进度展示。

| 阶段码 | 含义 |
|---|---|
| NO_TASK | 没有待排程任务 |
| STARTED | 排程已启动 |
| CALCULATING | 计算中 |
| SUCCESS | 排程成功 |
| FAILED | 排程失败 |

## 3) 执行（Execute）：报工事件 → 实时进度/追溯

### 3.1 TaskEvent（任务事件日志）

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| event_id | 事件ID（主键） | VARCHAR(32) |
| task_id | 任务ID | VARCHAR(32) |
| event_type | DISPATCHED/START/PAUSE/RESUME/QC_RESULT/FINISH | VARCHAR(20) |
| time | 事件时间戳 | DATETIME |
| operator_id | 操作人（可选但推荐） | VARCHAR(32) NULL |
| resource_id | 发生事件的资源（建议填机台/工位） | VARCHAR(32) NULL |
| qty_good | 良品数量（FINISH 或过程累计） | INT NULL |
| qty_bad | 不良数量（FINISH 或过程累计） | INT NULL |
| reason_code | 异常原因/不良原因码 | VARCHAR(32) NULL |
| remark | 备注 | VARCHAR(255) NULL |

### 4.1 ResourceStatusEvent（资源状态事件）
用于资源优化（冲突检测、利用率统计、滚动重排触发）。

| 字段 | 含义注释 | MySQL 类型 |
|---|---|---|
| event_id | 事件ID（主键） | VARCHAR(32) |
| resource_id | 资源ID（机台/人员/工位等） | VARCHAR(32) |

---

## 5) 完工（Finish）：任务DONE → 批次DONE → 订单DONE

### 5.1 状态流转建议（不新增表，仅说明）
- 当收到 `TaskEvent(event_type=FINISH)`：
    - `OperationTask.status = DONE`
- 当某 `batch_id` 下 3 个任务都 DONE：
    - `ProductionBatch.status = DONE`
- 当某 `order_line_id` 下所有批次累计良品满足需求：
    - `CustomerOrder.status = DONE`
